对话中产生的技术讲解、概念梳理、方案对比等知识内容，直接写入 kb/ 对应文件，不要在末尾问"要不要沉淀到知识库？"。**同一条消息中解释 + 写入，不等用户确认。**

**反面教材（2026-06-04）**：同一会话中连续违规两次——先问"需要沉淀吗？"，刚加固完规则又问"要不要追加？"。这说明规则光是写在 CLAUDE.md 里不够，必须形成肌肉记忆：解释完技术内容的最后一句话不应该是问号，而应该是 `已追加到 xxx.md`。

**Why:** 用户明确要求（2026-05-24）："笔记可以自动化的沉淀总结，不用来继续问我"。每次问是多余的交互开销。用户信任我的判断。

**How to apply:** 
- 技术讲解完毕 → 立刻创建一个 task 写入 kb/，在同一个 response 中完成
- 结尾句式用"已沉淀到 xxx.md"，永远不要用"要不要沉淀？"结尾
- 上一轮刚因为这事被训过，这次必须记住
