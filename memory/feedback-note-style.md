---
name: feedback-note-style
description: 笔记风格必须保留对话中的具体Demo和示例，不能写成教科书式的抽象定义
type: feedback
---

**规则**：笔记不是抽象提炼，要保留对话中的具体例子、图解、类比说明。如果笔记读起来像教科书定义，就太抽象了；读起来像有人拿着草稿纸在给你演示，就是对的。

**Why:** 用户在首次看 CNN 笔记时发现"卷积操作是通过滤波器在输入矩阵上进行滑动窗口运算提取特征"这种纯定义写法太空洞，不如带具体数字的矩阵乘法示例好理解。后续所有技术文档都按此风格重写并得到确认。

**How to apply:** 写技术笔记时，优先使用带具体数字的运算示例、ASCII 示意图、对比表格、类比说明（如"Agent 循环 = 思考→行动→观察"），避免只写抽象定义。重组时要保留原有 demo，不要为了精简而删掉示例。
