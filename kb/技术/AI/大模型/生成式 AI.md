# 生成式 AI：图片、视频、音频是怎么生成的

> 最后整理: 2026-05-06 | 来源: 多轮对话

## 一句话定位

大模型生成文本是"下一个词预测"，但图片/视频不能用这个方案——像素不是天然序列。主流方案是**扩散模型（Diffusion）**：先学怎么破坏，再反过来学怎么修复，修复就是在生成。

## 2026-05-06 - 图片、视频、音频的生成原理

### 为什么图片不能用"下一个词"的方式生成

```
文本生成（LLM 的做法）
━━━━━━━━━━━━━━━━━━━━━
"今天天气很" → 预测下一个词 → "好"
"今天天气很好" → 预测下一个词 → "，"

为什么能行？因为文字是天然的序列
一个字接一个字，时间线上走
```

```
图片生成不能这么做
━━━━━━━━━━━━━━━━━
图片不是"一个像素接一个像素"生成的
一张 512×512 的图片 = 262,144 个像素
每个像素有 256 种可能（RGB 各 8bit）

如果是"一个像素一个像素猜"：
  - 前面的像素猜错 → 后面全部崩
  - 生成一次要猜 26 万次，慢死

所以需要完全不同的生成范式
```

### 扩散模型（Diffusion Model）核心思想

**先学"怎么破坏"，再反过来学"怎么修复"——修复的过程就是在"生成"。**

#### 训练阶段：学破坏

```
第0步：一张清晰的猫图 🐱
       
第1步：加一点点高斯噪声 → 看起来还是猫，但有点糊
       
第2步：再加一点点噪声 → 勉强能看出是猫
       
...
       
第1000步：完全变成随机噪声 ⬛ (纯雪花点)

对于每一步，记录"这一步加了什么噪声"
然后训练一个神经网络：
  给它第 N 步的加噪图
  问它："你猜第 N-1 步应该长什么样？"

→ 训练几亿张图后，模型学会了"从噪声中还原"
```

#### 生成阶段：反向去噪

```
第1步：从纯随机噪声开始（一张雪花点图）⬛
       
第2步：让模型预测"这堆噪声里大概有什么"
       → 去掉一些噪声，留下一些轮廓（还是很糊）
       
第3步：再让模型预测下一步
       → 轮廓越来越清晰，"这里应该是猫的耳朵"
       
...

第1000步：一张清晰的猫图 🐱

关键：生成的猫不是训练集中任何一张的复制品
     而是模型从"噪声中想象"出来的全新猫
```

#### 类比

```
扩散模型 = 一个画家从白噪音中"看"出形状

过程就像：
1. 盯着电视雪花点（纯噪声）
2. 大脑开始幻想："这里面好像有只猫的轮廓"
3. 越看越清晰："这里有耳朵，那里有尾巴"
4. 最终"看"出了一张完整的猫
```

### 文字怎么指挥图片生成？——CLIP + 条件扩散

```
用户输入："一只穿西装的猫在办公室里"

LLM 把文字变成 embedding
CLIP 把文字 embedding 和图片 embedding 拉到同一个空间

扩散模型去噪时：
    每去一步噪，都问 CLIP：
    "我现在这个样子，和用户文字的描述匹配度提高了吗？"

    如果提高了 → 继续往这个方向去
    如果降低了 → 调整去噪的方向

相当于 CLIP 在旁边喊：
    "不对不对，用户说要穿西装的猫，你生成的像狗！"
    "对了对了，越来越像了，继续！"
```

```
┌──────────────┐         ┌──────────────┐
│ 纯随机噪声    │  ───▶   │ 扩散模型      │  ───▶  一张图片
│ (起点)       │         │ (U-Net)      │
└──────────────┘         └──────┬───────┘
                                ▲
                                │ 引导信号
                         ┌──────┴───────┐
                         │ CLIP / 文本   │
                         │ 编码器         │
                         └──────┬───────┘
                                ▲
                                │
                         "一只穿西装的猫在办公室里"
```

### 视频生成（Sora 那一类）

```
图片扩散:  2D 空间上去噪
           输入: 噪声图 (H × W × 3)
           输出: 清晰图 (H × W × 3)

视频扩散:  3D 空间上去噪（多了时间维度）
           输入: 噪声视频 (T × H × W × 3)
           输出: 清晰视频 (T × H × W × 3)

模型要同时保证：
  1. 每一帧本身是合理的（空间一致性）
  2. 帧与帧之间是连贯的（时间一致性）
```

#### Sora 的关键技巧：潜空间扩散

```
原始视频:  128帧 × 512×512 × RGB → 数据量巨大
↓ (VAE 压缩)
潜空间:     128帧 × 64×64 × 16维  → 小 64 倍

1. 在潜空间里跑扩散去噪（跟图片一样，但是 3D 的）
2. 去噪完成后，VAE 解码回像素级视频

为什么在潜空间做？
  - 计算量降 64 倍，训练和推理都快得多
  - VAE 已经学会了保留视觉信息，压缩不丢细节
  - 这叫"Latent Diffusion"，Stable Diffusion 也是这个思路
```

### 音频生成

```
音频跟文字一样，也是时序信号，所以做法更多样：

方案1：扩散模型（类似图片）
       加噪声 → 去噪声 → 生成音频波形
       代表: AudioLDM, Stable Audio

方案2：自回归预测（跟 LLM 一模一样）
       把音频分帧（每秒 50-100 帧），每帧当做一个"token"
       预测下一帧 → 预测下一帧 → ...
       代表: VALL-E（语音克隆）, AudioLM

方案3：Vocoder（声码器）
       先生成"语音特征"（梅尔频谱），再用声码器转成声音
       代表: Tacotron, WaveNet
```

### 两条路线正在融合

```
LLM 生成文本:  "给定开头，猜后面"
扩散模型生成图: "给定噪声，猜原图"

本质都是"猜"，只是:
  - LLM 猜的是离散的 token（有限词表里的一个词）
  - 扩散模型猜的是连续的向量（像素空间的还原方向）

融合方向:
  1. 把图片"token 化"（分成小块，每块编个号）
     → 用 LLM 的方式生成图片
     → Google VQGAN + Parti、Image GPT

  2. 给 LLM 加"扩散头"
     → LLM 直接生成图片
     → LlamaGen、Emu3

  3. 统一模型（text + image + video 全部一个模型）
     → 所有模态都 token 化，同一个 Transformer 预测
```
