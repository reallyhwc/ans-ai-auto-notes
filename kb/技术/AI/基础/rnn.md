# RNN (循环神经网络)

> 最后整理: 2026-05-04 | 来源: 多轮对话

## 一句话定位

专门处理序列数据（文本、时间序列、语音）的神经网络。核心是**"让上一次的输出成为下一次输入的一部分"**——没有卷积核，靠一套共享权重矩阵在时间维度上反复使用。

---

## 1. 直观理解：RNN 没有卷积核，靠什么？

```
CNN:  同一个卷积核扫过图片的每一块 → 空间上的参数共享
RNN:  同一套权重矩阵作用于序列的每一步 → 时间上的参数共享
```

两者都是"一套参数反复用"，但用的维度不同。CNN 的核对应"检测特定视觉特征"，RNN 的权重矩阵对应"当前词 + 之前记忆 → 新的记忆"这套规则。

RNN 的核心就三个矩阵，从训练开始到结束不变：

```
W_xh:  输入 → 隐藏状态   (把当前词压缩进记忆空间)
W_hh:  隐藏状态 → 隐藏状态 (把过去的记忆带过来)
W_hy:  隐藏状态 → 输出     (从记忆中提取当前需要的信息)
```

---

## 2. 核心机制：一步一步看

### 2.1 单步计算

```
时间步 t:

     x_t (当前词向量，比如"爱"的向量表示)
      │
      ▼
     ┌──── W_xh ────┐       ┌──── W_hh ────┐
     │ 把输入映射到   │       │ 把上一步记忆   │
     │ 隐藏空间       │       │ 传递过来       │
     └──────┬────────┘       └──────┬────────┘
            │                       │
            │       h_{t-1} ←───────┘
            │       (上一轮的隐藏状态)
            │            │
            ▼            ▼
     ┌─────────────────────────────┐
     │  h_t = tanh(W_xh·x_t + W_hh·h_{t-1} + b)  │
     │       ↑          ↑              ↑          │
     │    当前输入    过去记忆      偏置项        │
     └─────────────┬───────────────┘
                   │
                   ▼
              新的隐藏状态 h_t  ← 这一步"融合后的记忆"
                   │
                   ▼
          ┌─── W_hy ───┐
          │ 从记忆中     │
          │ 提取输出     │
          └─────┬───────┘
                ▼
              y_t (这一步的预测)
```

### 2.2 公式拆解：h_t = tanh(W_xh·x_t + W_hh·h_{t-1} + b)

这个公式就是 RNN 的全部。拆开来看，每一步都在做同一件事：**把新输入和旧记忆揉在一起，然后压扁到 (-1, 1)**。

#### Part A: W_xh · x_t — "把当前词翻译成隐藏空间的增量"

```
假设:
  词向量维度 = 3    (小到不真实，只为举例)
  隐藏状态维度 = 4  (隐藏状态通常比词向量大，给记忆留空间)

那么:
  x_t 是一个 3×1 的列向量，比如 "爱" = [0.2, 0.5, -0.3]
  W_xh 是一个 4×3 的矩阵

        W_xh (4×3)           x_t (3×1)       结果 (4×1)
  ┌                    ┐   ┌          ┐   ┌           ┐
  │  0.1   0.3  -0.2   │   │   0.2    │   │   0.23    │  ← 维度1的增量
  │ -0.4   0.1   0.5   │ × │   0.5    │ = │  -0.18    │  ← 维度2的增量
  │  0.2  -0.3   0.1   │   │  -0.3    │   │  -0.14    │  ← 维度3的增量
  │  0.5   0.2  -0.1   │   └          ┘   │   0.23    │  ← 维度4的增量
  └                    ┘                   └           ┘

  含义: "爱"这个词，在隐藏空间的4个维度上分别贡献了 0.23, -0.18, -0.14, 0.23
        每个维度代表一个"脑细胞"，W_xh 就是"听到这个词时各脑细胞的反应规则"
```

#### Part B: W_hh · h_{t-1} — "把旧记忆投射过来"

```
  h_{t-1} 是一个 4×1 的列向量，比如读完"我"之后的记忆 = [0.52, -0.31, 0.17, 0.08]
  W_hh 是一个 4×4 的方阵

        W_hh (4×4)                   h_{t-1} (4×1)    结果 (4×1)
  ┌                          ┐   ┌            ┐   ┌           ┐
  │  0.4  -0.2   0.1   0.3  │   │   0.52     │   │   0.31    │
  │ -0.1   0.5   0.2  -0.1  │ × │  -0.31     │ = │  -0.18    │
  │  0.3   0.1  -0.4   0.2  │   │   0.17     │   │   0.07    │
  │ -0.2   0.0   0.3   0.6  │   │   0.08     │   │  -0.03    │
  └                          ┘   └            ┘   └           ┘

  含义: 不是"原封不动搬运记忆"，而是把旧记忆做了个线性变换。
        W_hh 就是一个"记忆筛选器"——哪些记忆维度保留，哪些衰减，
        哪些和其他维度混合。
```

W_hh 是方阵的原因：隐藏空间维度不变——4维记忆进来，4维记忆出去。W_hh 决定了"记忆如何在四个脑细胞之间重新分配"。

#### Part C: 加法 + b — "新旧融合"

```
Part A 结果 (W_xh · "爱"):   [ 0.23]     ← "爱"带来的新信息
Part B 结果 (W_hh · 记忆):   [ 0.31]     ← "我"留下的记忆
偏置 b:                       [ 0.05]     ← 每个维度的默认基调
                              [ 0.00]
                              [-0.10]
                              [ 0.02]

逐元素相加:
  [ 0.23 + 0.31 + 0.05]   [ 0.59]
  [-0.18 + (-0.18) + 0.00] = [-0.36]
  [-0.14 + 0.07 + (-0.10)]   [-0.17]
  [ 0.23 + (-0.03) + 0.02]   [ 0.22]
```

**为什么加法有效？** 因为 W_xh 和 W_hh 是训练出来的——它们各自学会了：给定一个词给多少信号、给定旧记忆保留多少。加法就是"两股信号汇合"，不需要别的花活。

#### Part D: tanh() — "压扁"

```
tanh 函数:  输入任何实数，输出永远在 (-1, 1) 之间

  tanh(2.0)  ≈ 0.96
  tanh(0.59) ≈ 0.53
  tanh(0.0)  = 0.0
  tanh(-0.36) ≈ -0.35
  tanh(-2.0) ≈ -0.96

所以 [0.59, -0.36, -0.17, 0.22] 经过 tanh 变成:
     [0.53, -0.35, -0.17, 0.22]  ← 这就是 h₂，读完"我爱"后的新记忆
```

**为什么需要 tanh？** 没有它，h_t 的值会被 W_hh 反复乘，指数级爆炸。tanh 把每一步的输出硬压在 (-1, 1) 里，防止循环过程中数值失控。代价是——也导致了梯度消失（见 3.2）。

#### 完整合起来

```
h_t = tanh( W_xh·x_t + W_hh·h_{t-1} + b )
       ↑      ↑            ↑            ↑
       │      │            │            └─ 默认基调（每个维度始终偏一点）
       │      │            └─ 旧记忆经 W_hh 筛选后的信号
       │      └─ 当前词经 W_xh 翻译后的信号
       └─ 把加法结果压到 (-1,1)，防止爆炸
```

### 2.3 具体例子："我 爱 你"

```
t=1: x₁ = "我" 的词向量
     h₀ = [0, 0, 0, ...]  (初始记忆，全零)
     
     h₁ = tanh(W_xh · "我" + W_hh · [0,0,0...] + b)
        = [0.52, -0.31, 0.17, 0.08, ...]  
        ↑ 这就是读完"我"之后的记忆状态

t=2: x₂ = "爱" 的词向量
     
     h₂ = tanh(W_xh · "爱" + W_hh · [0.52, -0.31, 0.17, 0.08...] + b)
        = [0.61, -0.12, 0.34, -0.21, ...]
        ↑ "我"的记忆(通过W_hh) + "爱"的新信息(通过W_xh) → 新记忆

t=3: x₃ = "你" 的词向量
     
     h₃ = tanh(W_xh · "你" + W_hh · [0.61, -0.12, 0.34, -0.21...] + b)
        = [0.73, -0.08, 0.41, -0.15, ...]
        ↑ 包含了"我+爱+你"所有上下文信息

最后: y₃ = W_hy · h₃ → softmax → "这段文本最可能的情感是'正面'"
```

每一步用的是**同一套 W_xh, W_hh, W_hy**，不同的只是输入 x_t 和上一步的 h_{t-1}。

## 3. 训练：靠数据"堆"出来的，但不是简单的概率统计

### 3.1 训练方式 — BPTT（沿时间反向传播）

完全不是"统计词频和概率"，而是和 CNN 一样用**反向传播 + 梯度下降**。区别在于 RNN 要沿时间展开：

```
前向传播 (Forward):
  x₁ → h₁ → x₂ → h₂ → x₃ → h₃ → y₃ → 预测"正面"
                                          ↓
反向传播 (BPTT):                       误差：预测"负面"，错了！
  x₁ ← h₁ ← x₂ ← h₂ ← x₃ ← h₃ ← y₃ ← 误差信号沿时间往回传
```

误差从 t=3 一路传回 t=1，更新 W_xh、W_hh、W_hy。每一步都在调：**"怎么把新输入和旧记忆融合得更好"。**

### 3.2 朴素 RNN 的问题：梯度消失

```
"我出生在中国，...（300字后）...，所以我的母语是 ___"

BPTT 要从"母语"往回传误差到"中国"——
误差每往回传一步就乘一个小于1的因子，传 300 步后基本归零。
W_hh 关于"中国"的梯度 ≈ 0 → 学不到这个长距离依赖。
```

### 3.3 和 CNN 训练对比

| 维度 | CNN | RNN |
|------|-----|-----|
| 训练算法 | 反向传播 | BPTT（反向传播 + 时间展开） |
| 参数共享 | 空间共享（同一核扫全图） | 时间共享（同一权重每步复用） |
| 并行能力 | 天然并行（所有位置同时算） | 必须串行（h_t 依赖 h_{t-1}） |
| 数据处理 | 独立样本（每张图独立） | 依赖样本（每句话内部有顺序） |
| 学什么 | 核权重 — "什么特征有用" | W_xh, W_hh — "怎么融合输入和记忆" |

## 4. 进化：LSTM 和 GRU

朴素 RNN 的 tanh 激活函数导致梯度消失。LSTM 和 GRU 通过"门控"机制解决。

### 4.1 LSTM（长短期记忆）

在 RNN 基础上加了**三个门**和一条**独立的高速公路**：

```
RNN:              LSTM:
h = tanh(...)     ┌─ 遗忘门 (Forget Gate):  "旧记忆哪些要扔掉？"
                   ├─ 输入门 (Input Gate):   "新信息哪些要存？"
                   ├─ 输出门 (Output Gate):  "记忆的哪些部分要输出？"
                   └─ 细胞状态 (Cell State): 独立于隐藏状态，专门传长期记忆

                         细胞状态 C
                         ┌──────┼──────┐
                    ↙ 遗忘门  ↗  输入门  ↘ 输出门
                  (选择性删) (选择性存) (选择性输出)
                         │        │
                         └── 信息高速公路 ──┘
                          梯度不容易消失
```

### 4.2 GRU（门控循环单元）

LSTM 的简化版，两个门（重置门 + 更新门），效果相近但计算更快。

---

## 5. 关键特点

| 特点 | 含义 | 为什么重要 |
|------|------|-----------|
| 时间参数共享 | W_xh, W_hh 在每一步复用 | 无论序列多长，参数量固定 |
| 必须串行 | h_t 依赖 h_{t-1}，不能同时算其它步 | 训练慢，不能充分利用 GPU |
| 记忆衰减 | 越远的输入对当前影响越小 | 短序列 OK，长序列跑偏 |
| 变长输入 | 天然支持不定长序列 | 不像 CNN 需要固定尺寸输入 |

---

## 6. RNN 与 Transformer 的关系

RNN 的串行瓶颈是 Transformer 出现的主要动机。后者抛弃了"逐步传递记忆"的思路，改让所有词同时互相看（自注意力），解决了并行和长距离依赖两个问题。
