# 多模态 LLM（Multimodal LLM）

> 最后整理: 2026-05-06 | 来源: 多轮对话

## 一句话定位

多模态 LLM = 纯文本 LLM + "翻译器"（视觉编码器 + 投影层）。**大脑（Transformer）不变，只是输入通道从纯文字扩展到了图片/音频/视频。**

> 关联: [llm](./llm.md) — 纯文本 LLM 核心原理

## 2026-05-06 - LLM 与多模态 LLM 的区别

### 拿草稿纸画一下

```
LLM（纯语言模型）
━━━━━━━━━━━━━━━━━━━━
输入:  文本 token 序列
       ["你"、"好"、"，"、"今"、"天"、"天"、"气"、"怎"、"么"、"样"]
                ↓
        [Embedding → Transformer → 预测下一个 token]
                ↓
输出:  文本 token 序列

────────────────────────
只能处理文字，给它一张图片它完全"看不见"
```

```
多模态 LLM（MLLM / Multimodal LLM）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
输入:  文本 token + 非文本 embedding 拼接在一起

       文字:  ["这"、"张"、"图"、"里"、"有"、"什"、"么"、"？"]
                ↓              ↗
       图片:  [Patch1、Patch2、...、PatchN]  ← 视觉编码器把图转成向量
                ↓
        统一投影到同一个 embedding 空间
                ↓
        [多模态 Transformer → 预测下一个 token]
                ↓
输出:  文本（也可以输出图片/语音，取决于具体模型）

────────────────────────
同一套 Transformer，但输入通道从"纯文字"变成了"文字+图片/音频/视频"
```

### 具体怎么实现"看懂图片"的

以常见的视觉多模态模型（GPT-4V、Claude 图片理解）为例，关键多了一个**视觉编码器**：

```
第1步：切图
    ┌─────┬─────┬─────┐
    │ P1  │ P2  │ P3  │    把图片切成 N 个 patch（类似 CNN 的做法）
    ├─────┼─────┼─────┤
    │ P4  │ P5  │ P6  │    每个 patch 是一小块像素
    ├─────┼─────┼─────┤
    │ P7  │ P8  │ P9  │
    └─────┴─────┴─────┘

第2步：视觉编码（ViT - Vision Transformer）
    每个 patch → 经过 ViT 编码 → 得到一个向量
    [P1_vec, P2_vec, ..., P9_vec]

第3步：投影对齐
    把这些视觉向量乘一个线性层（投影矩阵）
    → 映射到和文字 token embedding 同一个空间

第4步：拼接
    输入序列变成：
    [文字 embedding..., 图片向量..., 文字 embedding...]

第5步：交给同一个 Transformer 处理
    自注意力机制会自动让文字 token 关注到图片的向量部分
    → 模型就"看懂"了图
```

**核心改动：在 LLM 前面加一个"翻译器"——把图片/音频等非文本内容翻译成向量，塞进原来只接受文字的位置，后面一切不变。**

### 类比

```
LLM         = 一个只读书的学生，文字理解能力极强
多模态 LLM  = 同一个学生，但给他配了一副眼镜（视觉编码器）
              和一副耳机（音频编码器），
              让他除了看书，还能看图、听声音。
              大脑（Transformer）还是同一个，
              只是输入通道多了。
```

### 能力对比

| 维度 | LLM | 多模态 LLM |
|------|-----|-----------|
| 输入 | 纯文字 | 文字 + 图片 + 音频 + 视频（取决于支持哪些模态） |
| 输出 | 纯文字 | 文字为主，部分模型可输出图片/语音 |
| 典型场景 | 写代码、翻译、对话、推理 | 看图回答问题、图片生成、语音对话、视频理解 |
| 代表模型 | GPT-3、LLaMA、Qwen（纯语言版） | GPT-4V、Claude（支持图片）、Gemini、Qwen-VL |

### 训练方式：适配器思路

多模态 LLM **不是从头训练**的。标准做法：

```
1. 拿一个已经训练好的 LLM（比如 GPT-4 / Claude）
2. 拿一个已经训练好的视觉编码器（比如 ViT / CLIP）
3. 用一个轻量级的"投影层"把它们连起来
4. 用图文配对的数据（图片 + 描述文字）做少量微调
   → 让模型学会"看图说话"

这叫"适配器"思路：大模型不动，加一个小配件就升级了
```
