- **[技术/AI]** 深度学习三大基础架构入门：CNN（图像）、RNN（序列）、Transformer（注意力机制），及与现代大模型的关系
- **[技术/AI]** 深入 CNN：分层特征（边缘→纹理→语义）是自然涌现的，不需要逐层人工标注——位置决定分工，反向传播自动搞定 → [CNN](../kb/技术/ai/cnn.md)
- **[技术/AI]** 深入 CNN 层1可解释性：层1特征能用数学语言精确描述（方向、频率、梯度），但无法用自然语言定义——因为日常语言为交流服务（造了"狗""花"），而卷积核为优化服务（产生的是"左暗右亮三像素竖边"这种没有命名需求的东西）。从 Layer 1 到 Layer N 是从数学语言到自然语言的渐变过程 → [CNN](../kb/技术/ai/cnn.md#35-层1到底识别出了什么数学语言-vs-自然语言)
- **[技术/AI]** 深入 RNN：重构成结构化文档。RNN 没有卷积核，靠三套共享权重矩阵（W_xh/W_hh/W_hy）在时间维度上反复使用。训练方式 BPTT（沿时间反向传播），与 CNN 一样靠反向传播+梯度下降，不是简单的概率统计 → [RNN](../kb/技术/ai/rnn.md)
- **[技术/AI]** 深入 RNN 公式拆解：h_t = tanh(W_xh·x_t + W_hh·h_{t-1} + b) 拆成四步（翻译输入→筛选记忆→新旧融合→压扁），每步带具体数字的矩阵乘法示例 → [RNN](../kb/技术/ai/rnn.md#22-公式拆解h_t--tanhw_xhx_t--w_hhh_t-1--b)
- **[技术/AI]** 深入 Transformer：重构成结构化文档。核心是"让每个词直接看到所有其他词"——自注意力 Q/K/V 机制、多头注意力、位置编码、Transformer Block 结构。详解为什么 Transformer 是大模型底座：并行性 + 长距离 O(1) 关联 + 架构简洁 → Scaling Law → [Transformer](../kb/技术/ai/transformer.md)
- **[技术/AI]** 深入 Transformer 计算复杂度：自注意力是 O(n²)——序列翻倍计算量翻四倍。为什么还是赢了 RNN？并行 O(n²) 在 GPU 上比串行 O(n) 快。现代优化：Flash Attention、滑动窗口、GQA、稀疏注意力 → [Transformer](../kb/技术/ai/transformer.md#9-计算复杂度on²-的代价)

- **[技术/Java]** 新增 Spring AI 笔记：ChatClient 流式（`stream().content()` 返回 `Flux<String>`）vs 非流式（`call().content()` 阻塞等待完整回复），底层都是同一个 HTTP POST 请求，区别在响应体传输方式（JSON vs SSE）→ [Spring AI](../kb/技术/java/spring-ai.md)
- **[技术/AI]** 补充 LLM 原理——HTTP SSE 流式传输：一字一字到屏幕的原理不是多次请求，而是一次 POST + `Transfer-Encoding: chunked` + `OutputStream.flush()`。每生成一个 token 立刻 push 到 TCP 发送队列，flush 本身几乎零成本 → [LLM #6](../kb/技术/ai/llm.md#6-http-sse-流式输出一字一字怎么传到你屏幕)
- **[技术/AI]** 补充 LLM 原理——GPU 推理吞吐量估算：推理分 Prefill（算力受限，快）和 Decode（显存带宽受限，慢）两阶段。A100 80GB 跑 70B 模型单卡约 1,500 t/s，Continuous Batching 让并发请求共享参数读取实现 batch 吞吐。DeepSeek MoE（671B/37B 激活）推理访存量减半，成本约为 70B 稠密模型的 50%，定价低但毛利率 50%+ → [LLM #7](../kb/技术/ai/llm.md#7-gpu-推理吞吐量与实际成本)
- **[技术/AI]** 更新 Agent/MCP 生态笔记——明确 Function Calling 是 LLM 输出 JSON 格式决策（`{"function_call": "getWeather", ...}`），并非真的执行函数。Skill = Prompt 片段 + 行为规则 + 工作流约束的结构化配置包，不具备 MCP 的执行能力。补充五者关系全景图：Agent（顶层决策者）、Skill（能力插件）、Prompt（大脑指令）、Function Calling（决策输出格式）、MCP（工具适配层）→ [Agent与MCP](../kb/技术/ai/llm-agent-mcp.md)
