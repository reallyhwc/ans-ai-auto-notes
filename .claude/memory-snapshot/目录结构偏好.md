**规则：kb/技术/AI/ 下使用 5 个并列子目录，最多保留这一规模——不允许再退化回单目录平铺。**

```
kb/技术/AI/
├── 基础/         (3 篇：cnn/rnn/transformer)
├── 大模型/       (8 篇：llm 核心 / mcp / 多模态 / 微调 / 部署 等)
├── Claude-Code/  (5 篇：架构 / 进阶工作流 / Harness / TDD skill / 远程操控)
├── AI-Coding/    (3 篇：CLI+IDE 工具对比 / 6 Level 路径 / 团队治理)
└── 应用/         (6 篇：Agent 开发 / OpenAI SDK / 客服 / 工作流平台 等)
```

**Why:** 2026-05-18 物理拆分前，应用生态/ 一个目录吞了 14 篇 5400 行，主题极度发散（Claude Code/AI Coding/Agent/LLM 应用全混一起）。用户明确选了"物理拆分"方案而非"frontmatter category 分组"——磁盘结构 = 视觉结构最不容易漂移。

**How to apply:**
- 新增 md 文件时按主题归类到 5 个子目录之一；不属于任何一个就讨论是否新建第 6 个（≥3 篇且边界清晰才新建，CLAUDE.md 有写规则）
- **不要**手改 INDEX.md 或 manifest.json——它们是 `build-index.js` 的产物。md 文件改完跑 `node scripts/build-index.js` 即可
- 跨子目录的 `> 关联:` 链接用相对路径 `../OtherSub/foo.md`（同子目录用 `./foo.md`）
- 任何子目录文件数超过 ~10 时考虑是否需要再拆
