# AI 驱动个人知识库 — 设计文档

## 概述

通过与 AI 对话自动构建个人知识库。AI 在回答问题之外，智能对内容进行分类、归纳、总结，逐步沉淀为结构化的 markdown 知识库。git + markdown 形式，主题分类与时间线双轨并行。

## 目录结构

```
ans-ai-auto-notes/
├── kb/                          ← 知识库主目录（按主题分类）
│   ├── 技术/                    ← 技术领域，子目录按具体技术方向
│   ├── 读书笔记/                ← 读书相关，一本书一个文件
│   ├── 日常思考/                ← 随笔、想法
│   └── action/                  ← 排查记录、好文摘要、技巧、灵感碎片
├── timeline/                    ← 按周归档的对话摘要
├── INDEX.md                     ← 总目录索引（分类聚合 + 标签）
├── overview.html                ← 可视化导览页
├── CLAUDE.md                    ← 项目指令文件（AI 行为规则）
└── .gitignore
```

### 文件组织规则

1. **同主题聚合到一个文件**：同一主题的知识点持续追加到同一个 md 文件，不按日期拆分
2. **文件内按时间倒序**：最新内容在文件顶部，`## YYYY-MM-DD - 标题` 作为二级标题
3. **子目录最多两层**：`技术/java/jvm-gc.md`
4. **新主题新建文件**：遇到全新主题时创建新文件

## 文件格式规范

### 知识文件（kb/ 下）

```markdown
# JVM GC

## 2026-05-15 - CMS 和 G1 的选型对比
> 来源: 对话 2026-05-15
> 关联: [[../../读书笔记/我看见的世界]]

（知识点正文...）

## 2026-05-01 - GC 日志关键参数解读
> 来源: 对话 2026-05-01

（知识点正文...）
```

### Timeline 文件（timeline/ 下）

按周生成，格式 `YYYY-WXX.md`，记录当周对话摘要并链接到对应主题文件的具体段落。

### INDEX.md

总目录索引，按分类列出所有主题文件及一句话描述。

## 更新策略（混合模式）

- **小知识点自动记录**：回答完问题后，自动提取知识点追加到对应主题文件
- **大改动主动提案**：文件拆分、合并、重组、目录结构变更时，主动向用户提案，确认后执行
- **主动性在 AI 侧**：不等用户下指令，自行判断时机并提案

## 跨文件关联

当一个知识点涉及多个维度（如读书中提到的技术概念），分别在对应文件中记录：
- 读书笔记侧重阅读上下文和感悟
- 技术文件侧重纯技术干货
- 两处互相留链接，形成关联网络

## 可视化导览（overview.html）

纯静态 HTML 页面，三 Tab 结构：
1. 分类导览 — 目录树，可展开折叠
2. 时间线 — 按周倒序的对话摘要
3. 标签/搜索 — 关键词检索

数据通过内嵌 JSON 维护，AI 在每次知识库更新时同步刷新。

## 会话工作流

```
提问 → 回答 → 自动提取知识点
                   ├─ 小知识点 → 自动写入主题文件 → 更新 timeline → 更新 INDEX
                   └─ 需大改动 → 提案 → 确认 → 执行
                        ↓
                  更新 overview.html 数据
                        ↓
                  会话结束 → git commit
```

## Git 规范

- 每次会话结束提交一次
- Commit message：`[YYYY-MM-DD] 会话主题摘要`
- 不自动 push，由用户决定

## 项目指令文件（CLAUDE.md）

独立维护 AI 行为规则，包括用户背景、知识库结构、更新规则、跨文件关联规则、Git 规则等。每次进入项目时读取。
